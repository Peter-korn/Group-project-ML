# Group-project-ML

### Преобразования датасета

* **id**: оставить в начальном датасете, но удалить перед обучением.
* **Месяцы**: синус-косинусная трансформация для сохранения информации о цикличности и близости месяцев.
* **Дата открытия**: One-Hot Encoding (OHE).
* **Торговая площадь**: One-Hot Encoding (OHE).
* **Населённый пункт** и **Регион**: Target Encoding с последующим масштабированием через Scaler.
* **Численность населения**, **Количество домохозяйств**, **Трафик пеший**, **Трафик авто**: Robust Scaler (выбран из-за сильного разброса данных и чувствительности к выбросам).
* **Количество объектов в радиусе**: метод главных компонент (PCA) для снижения размерности, затем масштабирование через Scaler.

### Модели для предсказания трафика и среднего чека

По каждой модели формируем по 36 предсказаний: для каждого из 12 месяцев в году сначала с лейблом «новый» по дате открытия для первого года, потом с лейблами «средний» и «старый» для прогнозирования устойчивых значений. Таким образом, получаем три полноценных годовых прогноза по основным показателям с учётом сезонности и новизны.

* **Линейная регрессия с Ridge-регуляризацией**
* **k-Nearest Neighbors (KNN)**
* **Random Forest**
* **CatBoost**
* **Multilayer Perceptron (MLP)**

### Модель для итогового предсказания успешности

* **Расчёт прогнозной выручки**: после получения помесячных прогнозов трафика и среднего чека агрегируется сначала месячная, а затем и годовая выручка. На выходе формируются три значения: прогнозная годовая выручка за первый год (лейбл `новый`) и за последующие устойчивые года (лейблы `средний` и `старый`).
* **Разметка исторического датасета (таргет)**: аналогичный расчёт проводится для существующих торговых точек из обучающей выборки — вычисляется их фактическая годовая выручка. Магазины разделяются на три группы (`новый`, `средний`, `старый`). Для каждой группы рассчитывается пороговый квантиль (например, 70-й перцентиль). Магазины, чья выручка превышает этот порог, размечаются как успешные (`1`), остальные — как неуспешные (`0`).
* **Интерпретация вектора предсказаний**: для оцениваемой локации строятся прогнозы по всем трём категориям. В результате формируется вектор успешности, дающий глубокую аналитическую оценку вместо бинарного исхода «ок / не ок»:
  * `[1, 0, 0]` — точка покажет отличный старт, но в долгосрочной перспективе показатели упадут.
  * `[0, 1, 1]` — проект на перспективу: на этапе открытия точка будет неликвидной, но затем выйдет на стабильную прибыль.
* **Выбор алгоритма**: в качестве базового решения применяется логистическая регрессия (Logistic Regression). При наличии вычислительных ресурсов возможно тестирование более сложных классификаторов.

> ⚠️ **Оптимизация вычислительной сложности:** в случае нехватки мощностей или сильного раздувания признакового пространства допускается исключение категории `старый` из пайплайна с сохранением логики для групп `новый` и `средний`.
